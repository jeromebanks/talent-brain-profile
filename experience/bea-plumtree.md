# BEA BID Division / Plumtree Software — Sr. Software Engineer

## Context

Contributed to various projects in various roles, including design, architecture, development and support. BEA acquired Plumtree Software in 2005.

## Contributions

- **Remoting Framework** – Evangelized an IoC POJO solution to implement remote services utilizing Spring, Lingo, and JMS. Investigated various remoting technologies for a SOA environment and certified a specific stack of open-source and proprietary packages as the organizational remoting solution. Developed a custom Spring namespace handler to expose standard remote services. Implemented a robust streaming solution for file transfers over JMS. Integrated OpenCounters (JMX service monitoring) into the remoting framework using Spring AOP interceptors. Prototyped a security integration with ACEGI and OpenSAML. Added JMS server reconnect logic and queue limits to Lingo and ActiveMQ.
- **Developer Tools / .NET Web Accelerator** – Ported the .NET Web Accelerator (for displaying .NET applications as a Plumtree Portlet) from .NET 1.1 to .NET 2.0, updating javascript rewrite libraries.
- **JSR-168 and WSRP Containers** – Provided support and fixed defects for Plumtree's JSR-168 portlet container solution.
- **Publisher / EDK API** – Extended internal native APIs for use as an external developer API. Added extensive unit test coverage.
- **Notification Server** – Designed and implemented extensions to a workflow notification service for email with MIME attachments. Used Velocity for email generation and Quartz for message scheduling.
